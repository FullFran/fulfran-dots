# My dotfiles (built on fulfran-dots)

Generated by [fulfran-dots](https://github.com/FullFran/fulfran-dots).

## Safer adoption pattern

Run the TUI to set up your machine:

```bash
nix run github:FullFran/fulfran-dots#tui
```

The TUI will ask which **config mode** you want:

| Mode | What it does |
|------|-------------|
| `safe` | Installs packages but does NOT overwrite any of your existing dotfiles. Recommended for your first install. |
| `full` | Ships all configs (overwrites existing dotfiles). Recommended for fresh installs. |
| `custom` | Asks you per config — you choose exactly which dotfiles to ship. |

Choose `safe` for your first install. The TUI generates `hosts/<hostname>.nix` with explicit `true`/`false` values for every toggle, so you can see exactly what is active and change it later.

If you want to manually opt in to specific configs after using `safe`, edit `hosts/<hostname>.nix`:

```nix
# hosts/my-host.nix
programs.fulfran = {
  core.enable = true;
  shell.enableBash = false;       # keep your .bashrc
  shell.enableZsh = false;        # keep your .zshrc
  tmux.enableConfig = false;      # keep your tmux.conf
  ghostty.enableConfig = false;   # keep your ghostty config
  nvim.enableConfig = false;      # keep your nvim
  lazygit.enableConfig = false;   # keep your lazygit
  btop.enableConfig = false;      # keep your btop config
  dev.enableGitHelpers = true;    # small helper function, safe to keep
  dev.enableJdk = false;          # opt-in to JDK 21 if needed
};
```

Once you are comfortable with a module, flip its toggle to `true` and run `home-manager switch` again.

See the [main README](https://github.com/FullFran/fulfran-dots#readme) for the full TUI flow documentation.

## Activate

```bash
nix run github:nix-community/home-manager -- switch --flake .#user@hostname -b backup
```

`-b backup` saves any overwritten files as `.backup` so you can recover them.
